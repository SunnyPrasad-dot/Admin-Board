Design a clean, modern, practical, and user-friendly admin dashboard UI for a photography booking and photographer management system.

IMPORTANT:
This is NOT a luxury marketing website.
This is an operational admin dashboard focused on clarity, workflow, usability, and management.

The UI should feel:
- modern
- organized
- professional
- practical
- easy to manage daily operations

Do NOT overdesign.
Avoid unnecessary animations or fancy visuals.
Focus on workflow, usability, readability, and management efficiency.

--------------------------------------------------

DESIGN STYLE:
- Clean modern dashboard
- Minimal and organized layout
- Soft shadows
- Rounded corners (10px–14px)
- Clear spacing and alignment
- Practical admin experience

--------------------------------------------------

COLOR SYSTEM:
- Main background: #F5F6FA
- Sidebar: #FFFFFF
- Cards: #FFFFFF
- Borders: #E5E7EB
- Primary accent: #4F46E5
- Secondary accent: #6366F1
- Success: #22C55E
- Warning: #F59E0B
- Danger: #EF4444
- Text primary: #111827
- Text secondary: #6B7280

Use subtle colors and maintain professional dashboard feel.

--------------------------------------------------

TYPOGRAPHY:
- Clean sans-serif typography
- Strong hierarchy
- Large page titles
- Minimal text
- High readability

--------------------------------------------------

DASHBOARD HOME PAGE:

TOP SUMMARY CARDS:
- Total Requests
- Confirmed Bookings
- Inquiry Requests
- Available Photographers

Below that:
Create 2 main tabs:

1. Confirm Requests
2. Inquiry Requests

--------------------------------------------------

CONFIRM REQUESTS SECTION:

Display booking requests in a clean table layout.

Columns:
- Client Name
- Event Type
- Event Date
- Location
- Status
- Assigned Photographer
- Action

Status badges:
- New
- Pending
- Assigned
- Confirmed

Actions:
- View Details
- Assign Photographer

Include:
- Search bar
- Filters:
  - By date
  - By event type
  - By status

IMPORTANT:
Confirm requests are users who completed booking flow and submitted booking confirmation.

--------------------------------------------------

INQUIRY REQUESTS SECTION:

This is for users who only submitted inquiry/contact details.

Display:
- Name
- Phone
- Inquiry message
- Date
- Action button

Actions:
- View inquiry
- Contact user

Inquiry requests should visually feel lower priority than confirmed requests.

--------------------------------------------------

SELECTED PACKAGE:

--------------------------------------------------

PHOTOGRAPHER ASSIGNMENT SECTION:

IMPORTANT WORKFLOW:

The system should NOT auto assign photographers.

Instead:
- System suggests available photographers based on selected date/time.
- Admin manually selects photographer.

Create 2 sections:

1. Suggested Available Photographers
2. Unavailable / Booked Photographers

AVAILABLE PHOTOGRAPHER CARD:
- Profile image
- Name
- Specialization
- Experience
- Availability badge
- Current workload
- Assign button

BOOKED PHOTOGRAPHER CARD:
- Faded/disabled appearance
- Non-selectable
- Show reason:
  "Booked on 22 May • Wedding Shoot"

IMPORTANT:
Do NOT completely hide unavailable photographers.
Show them faded with booking details.

Include:
- Search photographers
- Filter by:
  - Available
  - Busy
  - Assigned
  - By date

--------------------------------------------------

ASSIGNMENT FLOW:

Admin:
- opens request
- reviews details

- confirms assignment

Popup example:

Buttons:
- Cancel
